AI-基础学习

会话记忆

构建真正 智能、连贯、个性化 的对话系统的核心挑战之一

概述和作用

大模型本身是无状态的，我们给模型的输入有且只有 提示词（Prompt）。所以，大模型本身其实是不具备记忆能力的

但是看到的大模型看似是有记忆的，这是通过一定策略实现的。比如 把历史对话记录存储起来，每次调用大模型时，把历史对话记录取出来，再加入融合到 提示词中，使得大模型看似具有记忆。但这个记忆系统如何设计呢？

短期记忆和长期记忆

短期记忆

短期记忆的作用：

  1. 维护对话连贯性，记住当前对话轮次的上下文，确保模型能理解最新信息
  2. 执行负责任务的基础，在链式思考或工具调用中，上一步的结果是下一步的输入，这些都存储在短期记忆中

本质就是，记录最近几轮对话的内容，每次调用大模型时，将最近的几轮对话（User-Assistant对）直接拼接起来，再加入融合到 提示词中，一起送给模型

实现方式：

  1. 全量上下文，把最近轮次的对话，全部塞进 Prompt 里发给模型
    • 优点 保留的信息最完整
    • 缺点
      • 受限于大模型的上下文长度。由于上下文窗口不是无限的，当对话的轮次和内容越来越多，会超出上下文窗口限制，导致最前面的对话信息丢失（被动截断）
      • 塞进 Prompt 的记忆越多，大模型的调用成本也就越高、推理速度越慢
  2. 滑动窗口，只保留最近的 N 轮对话，更早之前的对话 主动 舍弃掉。（主动滑动）
    • 优点 能很好地控制住成本，提升推理速度 (因为保存的信息少了=>输入也变少了)
    • 缺点 因为只保留了最近几轮对话信息，之前的对话信息就丢弃了，可能影响模型输出结果的准确性。也无法回溯早期对话逻辑
  3. 滑动窗口+总结摘要，只保留最近的 N 轮对话，对 更早之前的对话生成一个简短的摘要存起来
    • 优点
      • 兼顾信息完整性与长度控制（近期原话 + 早期摘要）
      • 成本 / 推理速度可控（摘要压缩早期内容）
    • 缺点
      • 摘要过程会丢失细节信息（摘要也要依赖 LLM 生成摘要的质量）
      • 增加额外成本（调用 LLM 生成摘要）
      • 实现复杂度提升（需对接摘要接口 + 管理摘要与原话）

长期记忆

长期记忆的作用：

  1. 个性化，记住用户的偏好、身份信息（如“我对花生过敏”、“我是Java开发者”），使对话体验独一无二、个性化
  2. 知识持久化：存储从对话中提取的结构化事实（如“用户的项目是XX系统，使用SpringBoot”），避免用户重复陈述
  3. 跨会话关联：在用户多次访问中保持信息的连续性，实现真正的“认识你”的效果
  4. 减少冗余：无需在每次对话中重新介绍背景信息

指的是大模型系统通过外部机制实现的、能够跨多次对话会话持久化存储并调用用户相关信息的能力。它使AI能够记住用户在不同时间点提供的偏好、事实和历史交互，从而提供连贯、个性化且智能的服务，而非每次对话都从零开始

其核心在于将关键信息从临时的短期记忆中沉淀下来，形成一个可长期访问的个人知识库

本质就是一个独立于模型对话之外的、可读写的外部存储系统（如数据库、向量库）。它不是直接拼接到上下文里，而是在需要时被检索并注入到当前上下文中
