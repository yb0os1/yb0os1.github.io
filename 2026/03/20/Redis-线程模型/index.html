Redis-线程模型

redis为什么快

  1. 基于内存。redis是一种基于内存的数据库，数据存储在内存中，读写速度非常快；而且redis采用了高效的数据结构
  2. 执行命令是单线程模型。因为redis是基于内存，因此瓶颈可能是机器内存或网络带宽，而不是cpu，那么执行命令采用单线程模型，避免了多线程之间的竞争，省去多线程之间切换带来的时间和性能上的开销，而且还不存在死锁问题
  3. IO多路复用技术。在单线程的基础上，采用了I/O 多路复用技术，实现了单个线程同时处理多个客户端连接的能力，从而提高了 Redis 的并发性能

IO多路复用技术：一个线程处理多个IO流（select/epoll机制）。简单来说，redis只运行单线程的情况下，该机制允许内核中同时存在多个监听socket和已连接socket；内核会一直监听这些socket上的连接情况或数据请求，一旦有请求到达，就会交给redis线程处理，实现了一个redis线程处理多个IO流的效果

redis是单线程？多线程体现在哪

redis执行命令是单线程的，也就是 接收客户端请求、解析请求、执行读写操作、发送数据给客户端 这个流程是单线程的

但是redis程序并不是单线程的，多线程体现在：

  1. 关闭文件
  2. AOF刷盘
  3. 释放redis内存，lazyfree线程；执行unlink key/flushdb async/flushall async等删除操作的命令是交给后台线程执行（del是主线程处理）

此外，redis6.0引入了多线程IO，这个只针对发送响应数据，并不会以多线程的方式处理读请求

为什么这些是单独线程处理的？

  • 关闭文件、释放内存、AOF刷盘这些操作都是很耗时的，如果交给主线程处理那么很容易发生阻塞，这样就无法处理后续请求了

redis的IO多路复用

多路是指多个网络连接的客户端，复用是指复用同一个线程

IO多路复用技术其实就是一个线程处理多个IO流（select/epoll机制）。简单来说，redis只运行单线程的情况下，该机制允许内核中同时存在多个监听socket和已连接socket；内核会一直监听这些socket上的连接情况或数据请求，一旦有请求到达，就会交给redis线程处理，实现了一个redis线程处理多个IO流的效果

redis把多个客户端的连接socket注册到IO多路复用器中，由内核统一监听；哪个连接器有读写/连接事件，就把时间交给事件分派器，再由对应处理器执行具体逻辑

对上图详细描述：

  • 一个socket客户端与服务端连接时，会生成对应的一个套接字描述符，一个socket对应一个套接字描述符（文件描述符FD）
  • 多个客户端与服务端连接时，redis使用IO多路复用程序将客户端socket对应的FD注册到监听列表中。当客户端执行读写等命令时，IO多路复用程序会将命令封装成事件，并绑定到对应FD上
  • 文件事件处理器使用IO多路复用模块同时监听多个FD的读写情况，当产生读写、关闭、接收事件时，文件事件处理器将回调FD绑定的事件处理器进行处理相关命令

例如，：以 Redis 的 I/O 多路复用程序 epoll 函数为例。多个客户端连接服务端时，Redis 会将客户端socket 对应的 fd 注册进 epoll，然后 epoll 同时监听多个文件描述符(FD)是否有数据到来，如果有数据来了就通知事件处理器赶紧处理，这样就不会存在服务端一直等待某个客户端给数据的情形

redis网络模型

redis6.0之前使用的是单reactor单进程模式

所有工作都在同一进程中完成，实现简单，不需要考虑进程之间的通信，也不用担心多进程竞争

缺点

  1. 只有一个进程，无法充分利用多核CPU性能
  2. Handler对象在业务处理时，整个进程无法处理其他连接事件，如果业务处理耗时长，那么就会造成相应延迟

单reactor单进程不适用计算密集型场景，只适用于业务处理非常快速的场景

redis6.0之后，网络IO的处理改成了多线程（命令的执行还是单线程）
